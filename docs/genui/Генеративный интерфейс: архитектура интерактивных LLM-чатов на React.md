Для добавления графических и интерактивных элементов (кнопок, бегунков, графиков) в чат используется концепция Generative UI (GenUI). Этот подход позволяет большой языковой модели (LLM) выходить за рамки обычного текста и «генерировать» пользовательский интерфейс, адаптированный под контекст беседы.
Существует несколько основных архитектурных подходов и библиотек для реализации таких интерфейсов в React:

1.  Подход на основе вызова инструментов (Tool Calling)
    Это наиболее распространенный и стабильный метод.

        Механика: Вы предоставляете модели описание «инструментов» (например, getWeather или adjustParameter). Если модель решает вызвать инструмент, она возвращает структурированные данные (JSON).
        Рендеринг: На стороне клиента эти данные перехватываются и передаются в заранее написанный React-компонент.
        Интерактивность: Компоненты могут содержать кнопки или бегунки. Когда пользователь взаимодействует с ними (например, нажимает кнопку «Одобрить»), результат взаимодействия отправляется обратно модели через функцию addToolOutput или addResult, что позволяет продолжить диалог с учетом выбора пользователя.

2.  Использование специализированных фреймворков
    Vercel AI SDK (UI и RSC)

        AI SDK UI (рекомендуемый стабильный путь): Использует хук useChat, который получает поток данных от сервера. Вы итерируете по массиву сообщений и, если встречаете часть сообщения с типом tool-call, рендерите соответствующий компонент.
        AI SDK RSC (экспериментальный): Позволяет стримить готовые React Server Components напрямую с сервера на клиент с помощью функции streamUI. Это объединяет логику генерации и рендеринга, но на данный момент имеет ряд ограничений (например, мерцание компонентов).

assistant-ui и Tool UI
Это набор низкоуровневых примитивов (похожих на Radix UI), которые дают полный контроль над дизайном.

    Централизованный реестр: Все инструменты регистрируются через Tools() API.
    Библиотека компонентов Tool UI: Предлагает готовые интерактивные элементы, такие как Parameter Slider (бегунок параметров), Option List (список опций) и Approval Card (карточка подтверждения).
    Отложенный рендеринг: Позволяет ждать полной загрузки всех аргументов инструмента перед отображением сложного интерактивного компонента, чтобы избежать скачков интерфейса.

Gravity UI AIKit
Подход, ориентированный на презентационный слой и Atomic Design.

    Библиотека не зависит от конкретного AI SDK и фокусируется на визуальном представлении.
    Вы регистрируете свои компоненты в messageTypeRegistry, и чат отображает их, когда в потоке сообщений приходит соответствующий тип данных.

3.  Обработка интерактивных действий
    Чтобы добавить в чат кнопки и бегунки, которые действительно работают, используются следующие паттерны:

        Human-in-the-loop (Подтверждение пользователем): Перед выполнением опасного действия на сервере (например, платежа), модель отправляет запрос на одобрение. В чате появляется компонент с кнопками «Одобрить» и «Отклонить».
        Сбор ввода (User Input Collection): Вы можете создать форму или слайдер прямо в сообщении. После того как пользователь выставит значение и нажмет кнопку, компонент вызывает функцию, которая передает это значение обратно в LLM как результат выполнения инструмента.

4.  Управление состоянием (AI и UI State)
    Для сложных интерфейсов важно разделять состояние:

        AI State: Сериализуемый JSON (история сообщений), который понимает модель.
        UI State: Состояние на стороне клиента, где хранятся сами React-компоненты и их текущее визуальное состояние (например, текущее положение бегунка).

Рекомендация: Для создания чата с интерактивными бегунками и кнопками наиболее гибким вариантом будет использование assistant-ui в связке с библиотекой Tool UI, так как они предоставляют готовые примитивы именно для таких элементов управления.
